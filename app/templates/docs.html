1. Getting Started

SpecMint is a powerful tool that generates comprehensive test cases and artifacts from your OpenAPI specifications. Whether you're testing APIs manually or setting up automated testing pipelines, SpecMint provides the foundation you need.

What You'll Get

  • • Valid, boundary, and negative test cases
  • • JUnit/REST Assured test suites
  • • Postman collections
  • • WireMock stubs
  • • Realistic test datasets

2. Uploading Your OpenAPI Specification

Start by navigating to the Generate Test Cases page. You can upload your OpenAPI specification in two ways:

The main SpecMint interface for uploading and configuring test generation

File Upload

Drag and drop your OpenAPI specification file directly onto the upload area, or click to browse and select a file.

Supported formats: .json, .yaml, .yml

URL Input

Provide a direct URL to your OpenAPI specification if it's hosted online.

Example: https://api.example.com/openapi.json

📋 Supported OpenAPI Versions

SpecMint supports both OpenAPI 3.0 and 3.1 specifications, including all major features and extensions.

3. Configuration Options

Customize your test generation with these configuration options:

Test Cases per Endpoint

Choose how many test cases to generate for each API endpoint. More cases provide better coverage but take longer to generate.

Options: 5, 10, 15, 20, 50, 100, 200, 500 cases

Domain Hint

Help SpecMint generate more relevant test data by specifying your domain or industry.

Options: General, Pet Store, E-commerce, User Management, Financial, Healthcare, Social Media, Custom

AI Generation Speed

Balance between generation speed and test quality.

Fast (2-5s): Quick generation with good coverage
Balanced (5-10s): Optimal speed/quality balance
Best Quality (10-20s): Maximum test sophistication

6. Best Practices

✅ Do's

  • • Use the most recent version of your OpenAPI specification
  • • Provide domain hints for more relevant test data
  • • Start with 10-20 test cases per endpoint for initial testing
  • • Review generated tests before implementing in production
  • • Use the generated datasets as a foundation for your test environment

❌ Don'ts

  • • Don't rely solely on generated tests without review
  • • Don't use outdated or incomplete OpenAPI specifications
  • • Don't generate 500+ test cases unless you have specific coverage needs
  • • Don't skip the domain hint - it significantly improves test quality

7. Troubleshooting

Upload Issues

Problem: File upload fails or specification isn't recognized

Solution: Ensure your file is a valid OpenAPI 3.0 or 3.1 specification in JSON or YAML format. Check that the file isn't corrupted and contains valid OpenAPI syntax.

Generation Timeouts

Problem: Generation process takes too long or times out

Solution: Try reducing the number of test cases per endpoint or use the "Fast" AI speed setting. Large specifications with many endpoints may take longer to process.

Poor Test Quality

Problem: Generated tests seem generic or not relevant to your domain

Solution: Use the domain hint feature to specify your industry or domain. This helps generate more relevant test data and scenarios.

Download Issues

Problem: Can't download the generated ZIP file

Solution: Check your browser's download settings and ensure pop-up blockers aren't preventing the download. Try using a different browser if the issue persists.
